# TCP和UDP使用

TCP需要服务端和客户端

UDP即使客户端又是服务端：相当于发数据，直接使用UDP协议给指定的UDP接收端发数据就行，不需要向TCP提前建立连接。

## UDP代码

UDP 协议本身不分客户端和服务端，但在基于 UDP 的实际应用（如网络通信程序）中，通常会根据角色划分 “客户端” 和 “服务端”。
核心区别
协议层面：UDP 是无连接的传输层协议，通信前无需建立连接，数据以 “数据报” 形式直接发送，因此协议本身没有 “客户端”“服务端” 的概念，双方地位对等。
应用层面：实际开发中，为了实现特定功能（如数据请求 - 响应、文件传输等），会人为定义角色：
服务端：通常固定 IP 和端口，长期监听指定端口，等待接收其他设备（客户端）发送的数据，并按需返回响应（如 DNS 服务器、UDP 流媒体服务器）。
客户端：通常使用动态端口，主动向服务端的固定 IP 和端口发送数据（如向 DNS 服务器发送域名解析请求的设备）。



代码关键说明
核心前提：必须知道 UDP 服务器的 IP 地址 和 端口号（UDP 无连接，无法自动发现服务器）。
数据编码：字符串需转为字节数组（示例用 UTF-8，需与服务器编码一致，否则会乱码）。
阻塞与非阻塞：
示例中 Receive/ReceiveFrom 是阻塞方法（会等待直到收到数据），若需非阻塞，可使用 BeginReceive 异步方法或设置 Socket.Blocking = false。
异常处理：需捕获常见异常（如 IP 格式错误、端口被占用、服务器无响应等）。
资源释放：使用 using 语句自动释放 UdpClient 或 Socket，避免资源泄漏。
