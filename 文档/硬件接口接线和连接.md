

## Modbus相关

### RS485

RS485 多从站模块通常采用总线型拓扑结构进行接线，使用双绞线将主站与各从站的对应引脚连接，并在总线两端连接终端电阻，以下是具体接线方法：

1. **确认接口端子定义**：明确主设备和所有从设备 RS485 接口的 A+（或 D+、Data+、P+）、B-（或 D-、Data-、P-）和 GND（信号地）端子。
2. **连接信号线**：将主设备的 A + 端子连接到双绞线缆中的一根导线，然后把所有从设备的 A + 端子都并联连接到这根导线上。用同样的方法，将主设备和所有从设备的 B - 端子连接到双绞线的另一根导线上。
3. **连接地线（可选但推荐）**：将主设备的 GND 端子连接到线缆中的第三根导线，再把所有从设备的 GND 端子都连接到这根导线上，实现共地连接，以提高抗干扰能力。
4. **设置终端电阻**：在总线物理的两端，通常是距离主设备最远的两个设备端，在 A + 和 B - 之间并联一个 120 欧姆的终端电阻，用于吸收信号反射，防止信号畸变和数据错误。若设备内置可拨码开关启用 / 禁用的终端电阻，可通过设置开关来启用。
5. **处理屏蔽层（若使用屏蔽线）**：将屏蔽层在总线的一端，通常是主设备端，单点良好接地，另一端悬空，避免屏蔽层多点接地形成接地环路。

### RS232

RS232 主要用于点对点通信，若要实现多从站连接，需借助额外硬件或特定方法。具体接线方式如下：

* **点对点基本接线**：RS232 常用 DB9 连接器，最基本的通信只需连接 3 根线。将设备 A 的 TXD（发送数据，DB9 引脚 3）与设备 B 的 RXD（接收数据，DB9 引脚 2）相连，设备 A 的 RXD 与设备 B 的 TXD 相连，然后将两个设备的 GND（信号地，DB9 引脚 5）连接在一起。
* **多从站接线 - 利用多串口卡或串口服务器**：在主机上安装多串口卡，如 PCIe 转 8 串口卡，每个串口通过上述点对点的方式分别连接一个从站设备。或者使用串口服务器，将 RS232 转换为以太网信号，从站设备也通过串口连接串口服务器，通过 TCP/IP 网络实现多设备组网。
* **多从站接线 - 通过 RS232 转 RS485 转换器**：先将 RS232 设备连接到 RS232 转 RS485 转换器，再将多个转换器的 RS485 端按 RS485 总线的接线方式连接，即把所有转换器的 A + 连接在一起，B - 连接在一起，还可在总线两端连接 120Ω 终端电阻，以增强信号稳定性。
* **多从站接线 - 使用串口集线器**：将主机和各个从站的 RS232 接口都连接到串口集线器相应端口，由集线器内部采用轮询或时分复用技术来模拟多点连接，实现多从站与主机的通信。

### Modbus TCP

#### Modbus TCP 多从站接线及网络架构

Modbus TCP 基于以太网进行通信，以下是其接线和网络架构相关要点：

##### 设备准备

1. **主站设备**：通常是工业控制器、SCADA 系统或带有 Modbus TCP 主站功能的 PC 等。这些设备需要具备以太网接口。
2. **从站设备**：各种支持 Modbus TCP 协议的智能设备，如 PLC、传感器、仪表等，同样需要以太网接口。

##### 网络拓扑

1. **星型拓扑**：这是最常见的拓扑结构。使用以太网交换机作为中心节点，主站和各个从站分别通过网线连接到交换机的不同端口。这种拓扑结构易于维护和扩展，当某个从站出现故障时，不会影响其他从站与主站的通信。
2. **总线型拓扑（不常见但可行）**：主站和从站依次连接在同一条以太网线缆上，两端需要连接终端电阻（通常为 50Ω - 100Ω）以防止信号反射。但由于以太网设备大多设计为星型连接，总线型拓扑在实际中应用较少。

##### 接线步骤

1. **网线连接**：使用标准的以太网线（如 Cat5e 或 Cat6），一端插入主站设备的以太网接口，另一端插入交换机的一个端口。
2. **从站连接**：对每个从站设备重复上述操作，将从站的以太网接口通过网线连接到交换机的其他端口。确保所有设备连接牢固，指示灯正常亮起，表示物理连接正常。

##### IP 地址设置

1. **主站**：为其设置一个固定的 IP 地址，确保该 IP 地址与从站设备处于同一子网内。例如，如果从站设备的 IP 地址范围是 192.168.1.10 - 192.168.1.20，主站可以设置为 192.168.1.1。
2. **从站**：每个从站设备都需要设置唯一的 IP 地址，并且要与主站处于同一子网。一些设备可以通过设备自身的设置界面（如 Web 界面、本地配置软件等）来设置 IP 地址，也有些设备可能通过拨码开关等硬件方式设置。

##### 端口设置

Modbus TCP 使用固定端口 502，主站和从站设备都需使用此端口进行通信。一般无需额外配置端口，只要网络连接和 IP 地址设置正确，设备间就能通过 502 端口进行 Modbus TCP 通信。

##### 开发过程中的考虑因素

1. **寻址**：在代码中，通过从站设备的 IP 地址和端口号（502）来定位每个从站。在读取或写入数据时，要指定对应的从站地址。
2. **并发通信**：若要同时与多个从站通信，需考虑使用多线程或异步编程技术，以避免通信过程中的阻塞，提高通信效率。
3. **错误处理**：网络通信可能会出现各种错误，如连接超时、数据传输错误等。在开发中要加入完善的错误处理机制，确保程序的稳定性和可靠性。
   
   

### 总结

一般多从站使用RS485比较方便，通过主站的A+和B-两个线直接并联其它多个从站设备上，从站设备设置不同的站号，主站就可实现不同的从站读取。

RS232就是用于当个设备之间的互相通讯，多从站学要转换器转接成RS485才能多从站。

## 模拟量接口

### 电压型模拟量接口

* **单极性电压输入输出**：对于单极性的电压信号，如 0 - 10V 。常见的接口形式有正端（V+ ）和公共端（COM ）。输入时，信号源的正极连接到模块的 V+ 端，负极连接到 COM 端；输出时，模块从 V+ 端输出电压，COM 端为参考地，电流从 V+ 经负载流回 COM 端。所以这种情况下不是正负一对接口，而是一个信号端和一个公共端。
* **双极性电压输入输出**：比如 -10V - +10V 的信号，通常会有正端（V+ ）和负端（V- ）两个接口，以及一个公共端（COM ）。输入时，信号源的正极接 V+ ，负极接 V- ；输出时，V+ 输出正电压，V- 输出负电压，公共端作为电压参考基准。这种情况下可认为有类似正负一对的接口用于接收或输出双极性电压信号。

### 电流型模拟量接口

* **两线制电流信号**：常见于 4 - 20mA 电流环，一般只有两根线。其中一根线用于供电，另一根线用于传输电流信号。这种接口没有明显的正负之分，因为电流是在这两根线构成的回路中流动。传感器从这两根线获取电源，并将测量值以电流形式反馈，接收端（如 PLC 模拟量输入模块）在回路中检测电流大小。
* **四线制电流信号**：通常有两根线用于供电（V+ 和 V- ），另外两根线用于传输电流信号（I+ 和 I- ）。在这种情况下，对于电流信号传输部分，可看作有正负一对接口（I+ 和 I- ），电流从 I+ 流出，经负载流回 I- ，实现模拟量电流信号的传输。
